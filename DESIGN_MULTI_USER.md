# 多用户持仓匹配发送 设计概要

## 核心功能
1. 支持维护多个用户的持仓配置。
2. 每个用户有自己的基金组合和收件邮箱。
3. 系统一次运行时，按用户逐个生成个性化报告。
4. 每份报告只发送到该用户绑定的邮箱。
5. 保留现有新闻抓取、日志、定时任务和 Markdown 存档能力。

## 主要文件
- `users.json` - 多用户配置入口
- `portfolio.py` - 读取单用户或多用户持仓数据
- `generate_report.py` - 循环生成多用户报告
- `send_email.py` - 支持按用户发送对应报告
- `DESIGN_MULTI_USER.md` - 当前设计文档

## 技术选型
- JSON 作为多用户配置格式
- Python 逐用户处理
- 兼容 OpenAI 接口的大模型做个性化分析
- Markdown 继续作为每位用户的报告输出格式

## 第一阶段范围
1. 用户信息和持仓统一放在 `users.json` 中。
2. 每个用户配置：姓名、邮箱、持仓列表。
3. 系统一次运行时，为每个用户单独生成报告和发送邮件。
4. 暂不做图形界面、数据库和权限系统。

## 原子功能点拆分
1. 增加多用户配置文件和读取逻辑。
2. 本地输出可验证的多用户摘要。
3. 逐用户生成报告并命名区分。
4. 邮件发送改为按用户目标邮箱发送。
5. 更新文档和定时执行说明。
